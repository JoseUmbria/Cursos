Resumen curso de Git / GitHub

Git es un software de control de versiones de un proyecto que facilita su desarrollo y mantenimiento de manera individual o como trabajo en equipo. Guarda un registro de todos los cambios y se puede recuperar información anterior si es necesario.

Es posible trabajar con múltiples archivos, además de permitir el trabajo de varias personas al mismo tiempo. Para ello se usan las ramas, que posteriormente se fusionan con merge.

El programa GIT se debe instalar y posteriormente se trabajará con GIT BASH que es la consola donde se desarrollarán todas las funciones.

En GIT hay dos áreas que son el área de ensayo y el repositorio local. La primera es temporal y se puede ver si el fichero tiene seguimiento de cambios por GIT, la segunda es la copia local del archivo.

Para trabajar con GIT Bash puedo directamente hacerlo en la carpeta donde se guarda la copia de respaldo y así directamente está en el directorio correspondiente.

Funciones importantes de GIT

git init -> Se ejecuta sólo una vez al empezar el proyecto o empezar el seguimiento del proyecto.

git add -> añade un archivo al área de ensayo (internamente y se muestra con una A verde).

Ejemplo : git add index.html

git commit -> lleva el archivo del área de ensayo al repositorio local y se le puede asignar nombre para saber qué se ha hecho.

Ejemplo : git commit -s "descripción"

git status -s -> dice los archivos que hay en el área de ensayo de nuestro proyecto.

git log --oneline -> permite ver las copias de respaldo que se han hecho con sus comentarios.

git reset --hard [codigo instantáneo] -> Permite volver a la versión anterior, pero piero la versión que haya deshecho.

Cuando hago cambios en un fichero que ya estaban en el repositorio local vuelve a aparecer con git status -s. Esta vez aparece con M roja y tenemos que añadir al área de ensayo y posteriormente al repositorio local.
